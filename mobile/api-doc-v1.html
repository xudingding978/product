API Documentation for Business Directory Environment

WebService Search Parameter Defitions

There are essentially 3 different types of searches that can be performed via this API. Each of the types are shown in the examples below with their valid parameters

Search Types

stype = search type, this defines the correct set of parameters to pass
valid search types are:
  • loc
    • location based search, requires lat and lng (number)
  • reg
    • region based search, requires a region string (string)
  • cat
    • category based search, requires a starting category id (number)
  • sup
    • list information on suppliers, requires a supplier id (number)

Example of Search By Near Me

http://m.tonz.co.nz/v1/?stype=loc&lat=-36.847385&lng=174.765735&rad=100&drec=6&maxr=10


stype = loc
lat = should be the latitude of the users loction
lng = should be the longitude of the users loction
rad = radius to be used to show the radius in which to show
drec = the directory ID for the query to enquire into ( 6 = TONZ directory)
maxr = maximum number of results to return for any query
page = the page of records to return

Example Location we are using is 8 Wyndham St, Auckland
-36.847385, 174.765735

Example of Search By Category

http://m.tonz.co.nz/?stype=reg&crec=3595&drec=6&maxr=10&page=1


stype = cat
drec = the directory ID for the query enquire into
maxr = maximum number of results to return for any query
page = the page of records to return


Example of Search By Region

http://m.tonz.co.nz/v1/?stype=reg&drec=6&maxr=10&page=3&sreg=auckland


stype = reg
drec = the directory ID for the query enquire into
maxr = maximum number of results to return for any query
page = the page of records to return for a page or show more
sreg = string of the region that search will query for
